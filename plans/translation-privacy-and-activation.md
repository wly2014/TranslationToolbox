# Plan: 启动激活、默认外发与长句 Key 门槛

> **实现状态**：已在 **v0.6.0** 落地（见 `CHANGELOG.md`）。下列验收标准已勾选。

> Source PRD: (1) 默认「启动即激活」；(2) 长句（大模型路径）须配置有效 Key 方可请求；(3) 分期策略已调整：**Phase 1 为「启动即激活 + 默认外发」**（开箱即用，与当前「默认可用第三方接口」一致）。原「默认零外发」方案不再作为 Phase 1 目标；若后续需要「可关闭第三方」，可在设置层单独立项（见架构说明）。

> **用户确认（2026）**：分期粒度合适；Phase 1 不拆两期；**原 Phase 2 与 Phase 3 合并**为一期交付。

## Architectural decisions

适用于后续各阶段的稳定决策：

- **激活模型**：使用 VS Code **工作区启动完成后激活**（`onStartupFinished`），不依赖用户先执行命令；**最低 VS Code 引擎版本**需满足该激活事件。
- **默认外发策略**：**默认允许**将选中文本发往第三方服务（有道、火山方舟等）以返回翻译，保证安装后无需额外开关即可划词翻译；若产品后续需要合规「关闭外发」，通过 **单独配置项** 关闭所有外网翻译（与 Phase 1 默认行为可并存为后续小版本）。
- **路由与门槛**：短词/词典类与长句/大模型类仍按现有规则分流；**长句路径在发起大模型请求前必须校验 API Key**（非空、非占位默认值），否则仅 Hover 提示，**不**请求大模型网关。
- **配置契约**：豆包 API Key、模型标识等以 **统一命名空间下的工作区配置**暴露；兼容历史用户已保存的旧键名（实现时确定回退读取顺序）。
- **可观测与测试**：无效 Key、网络错误等路径可在自动化测试中通过替身验证，避免依赖真实外网。

## Phase 1: 启动即激活 + 默认外发

**User stories**: 扩展随 VS Code 启动完成即可加载；**默认**即可划词并向外发起翻译请求（短词走词典、长句走大模型，与当前产品预期一致），无需先执行快捷键激活。

### What to build

从「打开窗口」到「划词悬停」的完整路径：扩展在启动完成后自动激活并注册 Hover；**默认配置下**短词、长句均可按现有逻辑调用第三方（本阶段可**暂不**强制长句 Key 校验，若与 Phase 2 同时实现则一并验收）。核心交付物是 **清单中的激活事件改为启动完成即激活**，且默认行为为**允许外发**。

### Acceptance criteria

- [x] 加载工作区后，扩展无需用户先执行快捷键/命令即已激活（可经扩展主机日志或等价方式验证）。
- [x] 默认配置下，划词悬停可对第三方服务发起请求并得到结果（与「默认外发」一致）。
- [x] 扩展测试与 Lint 通过（或按新激活方式更新最小断言）。

## Phase 2: 短词外发路径 + 长句必须有效 Key（合并原 Phase 2/3）

**User stories**: 短词路径稳定走第三方词典类接口；长句路径仅在 **有效** 豆包 API Key 存在时请求大模型，否则提示配置且不发送选文至大模型网关。

### What to build

在同一垂直切片内端到端验收：**短词** → 有道等请求 → Hover 展示；**长句** → 校验 Key → 有效则调用大模型，无效则 Hover 说明且不请求；弱网/HTTP 错误有明确提示且不导致扩展宿主崩溃；配置中的模型标识与实际请求一致（可与 Phase 3 部分重叠，但以本阶段验收为准）。

### Acceptance criteria

- [x] 短词悬停可完成第三方词典类请求并展示结果（失败时有可感知提示）。
- [x] Key 无效或占位时，长句悬停**不**发起大模型请求，且 Hover 提示用户配置有效 Key。
- [x] Key 有效时，长句可完成端到端大模型翻译（真实或替身环境可验证）。
- [x] 错误处理不因缺失响应体等导致扩展崩溃。

## Phase 3: 配置一致性、兼容与文档

**User stories**: 旧版设置可读；README/快速入门与「启动激活、默认外发、长句需 Key」一致；贡献者可知如何验证。

### What to build

统一配置键与运行时读取；旧键名回退；更新用户文档与必要 Release Notes；`DouBaoModel`（或等价项）与请求体一致。

### Acceptance criteria

- [x] 界面可选模型与实际请求模型一致。
- [x] 仅保留历史 Key 字段的用户仍可正常使用（或文档说明迁移步骤）。
- [x] README / 快速入门反映当前激活方式与长句 Key 要求。
- [x] Lint 与扩展测试通过。
